## 1. プロジェクトの背景と目的

### 1.1 背景

生成AI（PerplexityAI、ChatGPT Search、Claude、Gemini等）によるWebサイトのスクレイピングが2024年以降、指数的に増加しています。これらのAIシステムは：

- `robots.txt` を無視して大量クロールを実施
- コンテンツをAI回答の学習・生成に無断利用
- サイトオーナーに一切の対価を支払わない

**Cloudflare社は2025年に「Pay-per-Crawl（PPCW）」機能を正式発表。** これにより、AIクローラーに対してHTTP 402を使ったMachine-to-Machine（M2M）自動決済が技術的に実現可能になりました。

本プロジェクトは、この Cloudflare PPCW を活用した**日本初のAIアクセス課金インフラ「Adctor」**の構築を目的とします。

### 1.2 事業目的

1. **防衛と収益化:** 独自コンテンツへの無断AIアクセスを防ぎ、1アクセスあたりのマイクロペイメントで収益化
2. **AI検索最適化（GEO）:** 戦略的に無料開放するページには構造化データを付与し、AI検索エンジンからの認知を最大化
3. **データ資産の可視化:** どのAIが・どのコンテンツを・どれだけ求めているかを定量把握
4. **日本市場での先行者優位:** PPCWが普及する前に市場を確保

### 1.3 ビジョン

> 「有益な情報資産をAI検索で守り、管理し、運用する——攻めのサイバーセキュリティマーケティング」

日本円の換金力と世界の有効通貨をカバーするAI課金システムを通じて、日本のコンテンツ事業者が世界のAI企業から正当な対価を受け取れる仕組みを構築します。

## 2. システムの解決課題

| 課題 | 現状の問題 | Adctorによる解決 |
|------|-----------|----------------|
| AI無断クロール | robots.txtを無視してデータを収集 | Cloudflare Edgeでリアルタイム検知・ブロック |
| 機会損失 | データが無料でAI学習に使われる | 1アクセスあたり課金（¥1〜¥100） |
| データ不可視 | どのAIが何を見ているか不明 | Logpush→S3→Athenaで全ログ可視化 |
| 価格設定困難 | コンテンツ価値に応じた価格設定ができない | ディレクトリ単位のダイナミックプライシング |
| AI偽装対策 | UA偽装でAIが人間になりすます | IP・挙動・フィンガープリント複合判定 |

## 3. 希望するシステム構成

```
[AIクローラー]
      ↓ HTTP Request
[Cloudflare Edge Layer]  ← Bot Management + Pay-per-Crawl
      ↓ AI検知 → HTTP 402
[Stripe決済 (M2M)]
      ↓ 決済完了
[AWS オリジンサーバー]   ← EC2/ECS + RDS + S3
      ↓ コンテンツ配信
[Logpush → S3 → Athena] ← ログ蓄積・分析
      ↓
[管理画面 (React/Next.js)] ← ダッシュボード・価格設定
```

### 3.1 技術スタック（希望）

| レイヤー | 技術 | 役割 |
|---------|------|------|
| CDN/Edge | Cloudflare Workers + Bot Management | AIクローラー検知・PPCW実行 |
| バックエンド | FastAPI (Python) / Node.js | API・ビジネスロジック |
| フロントエンド | Next.js 16 + TypeScript + Tailwind CSS | 管理画面・ランディングページ |
| 決済 | Stripe Connect | M2M決済・Payout管理 |
| データベース | Amazon RDS (PostgreSQL) | 価格マスタ・ユーザー管理・トランザクション |
| ログ分析 | AWS S3 + Athena | Cloudflare Logpushの蓄積・分析 |
| メール通知 | Resend API | 決済通知・レポート配信 |
| 画像素材 | Unsplash API | 管理画面・LPの素材取得 |
| IP判定 | IPinfo API | クローラーの地理情報・ASN判定 |
| アラート | Slack Webhook | リアルタイム異常検知通知 |

## 4. プロジェクトスコープ（開発範囲）

### Phase 1: インフラ基盤構築（必須）
- [ ] Cloudflare Workers のデプロイ・Bot Managementルール設定
